1 · HashMap / HashSet — O(1) Lookup
Two Sum · Group Anagrams · LRU Cache · Consecutive Sequence
Easy/Med ▼
Concept Questions
Problems & Links
Key Code
What an interviewer might ask before coding
What is the time complexity of HashMap.get() and HashMap.put()? O(1) average. O(n) worst case due to hash collisions. Java HashMap uses chaining — collision means traversing a linked list (or red-black tree if chain > 8).
HashMap vs HashSet — what's the difference? HashSet is backed by a HashMap where all values map to a dummy constant. Use HashSet when you only care if something EXISTS. Use HashMap when you need to store associated data (index, count, etc).
When would a HashMap cause O(n) lookups? All keys hash to same bucket (degenerate input or poor hash function). Java 8+ converts chains > 8 to red-black trees to cap at O(log n).
Why does LRU Cache need a HashMap AND a doubly linked list? HashMap gives O(1) key lookup. Doubly linked list allows O(1) removal of any node (because you have prev/next pointers) and O(1) insertion at head. Neither alone gives O(1) get+put.

// Two Sum — O(n) time, O(n) space
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int comp = target - nums[i];
        if (map.containsKey(comp)) return new int[]{ map.get(comp), i };
        map.put(nums[i], i);   // store AFTER check (handles duplicates)
    }
    return new int[]{};
}

What makes sliding window O(n) instead of O(n²)? Both left and right pointers only move forward — never backward. Each element enters the window once (right++) and leaves once (left++). Total moves ≤ 2n.
Fixed vs variable window — how do you know which? Fixed: "max sum of K consecutive elements" → window size is constant. Variable: "longest substring with at most K distinct chars" → window grows/shrinks based on condition.
When do you shrink the left pointer? When the window violates the constraint (duplicate found, too many chars, sum exceeds target). Shrink until the constraint is satisfied again, then continue expanding right.

// Longest Substring Without Repeating — O(n)
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> last = new HashMap<>();
    int left = 0, max = 0;
    for (int r = 0; r < s.length(); r++) {
        char c = s.charAt(r);
        if (last.containsKey(c) && last.get(c) >= left)
            left = last.get(c) + 1;   // jump left past duplicate
        last.put(c, r);
        max = Math.max(max, r - left + 1);
    }
    return max;
}

Medium ▼
Concept Questions
Problems & Links

Concept questions
DFS vs BFS — when to use each on trees? DFS (recursion/stack): path problems, depth, subtree properties, all-paths. BFS (queue): level-by-level processing, shortest path, level order output, nodes at distance K.
Inorder, Preorder, Postorder — what's the visit order? Inorder = Left→Node→Right (gives sorted order for BST). Preorder = Node→Left→Right (serialize tree, copy tree). Postorder = Left→Right→Node (delete tree, calculate sizes bottom-up).
BST property — what guarantees does it give? Left subtree: all nodes < root. Right subtree: all nodes > root. Inorder traversal of BST = sorted array. Search, insert, delete = O(h) where h = height (O(log n) balanced, O(n) worst).
How do you validate a BST? Don't just check node > left child and < right child (that's wrong for subtrees). Pass min/max bounds down: every node must satisfy min < node.val < max.

// Level Order BFS — must know this template
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size();       // snapshot — how many nodes at this level
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = q.poll();
            level.add(node.val);
            if (node.left != null) q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        res.add(level);
    }
    return res;
}

How does Floyd's cycle detection work? Slow pointer moves 1 step, fast moves 2. If cycle exists they meet. No cycle → fast reaches null. After meeting: reset slow to head, both move 1 step at a time → meet at cycle start.
Why use a dummy head node? Simplifies edge cases where the head itself might be removed (e.g., Remove Nth from End when n = list length). Dummy.next is always the real head — no special case needed.
How do you reverse a linked list in-place? What three pointers do you need? prev (null), curr (head), next (save before overwriting). Reverse the link, advance. Return prev (new head).

// Reverse Linked List — must code cold in 3 minutes
public ListNode reverseList(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next; // save before overwriting
        curr.next = prev;          // reverse link
        prev = curr;               // advance prev
        curr = next;               // advance curr
    }
    return prev;                   // new head
}

5 · Binary Search — Not Just Sorted Arrays
Rotated Array · Koko Bananas · Capacity to Ship · First Bad Version
Medium ▼

Concept questions
Why use left + (right-left)/2 instead of (left+right)/2? Overflow prevention. If left=1B and right=2B, (left+right)/2 overflows int. left+(right-left)/2 always safe.
When do you use left < right vs left <= right? left <= right: standard search (exit when not found). left < right: searching for boundary/minimum (exits when left==right, which is the answer).
What is "binary search on the answer"? Instead of searching an array, search the space of possible answers. "Minimum eating speed" → binary search between 1 and max(pile). For each candidate speed, check if valid in O(n).
Rotated sorted array — key insight? One half is ALWAYS sorted. Check which half. If target is in the sorted half → search there. Otherwise search the other half.

// Koko — binary search on answer
public int minEatingSpeed(int[] piles, int h) {
    int left = 1, right = 0;
    for (int p : piles) right = Math.max(right, p);
    while (left < right) {
        int mid = left + (right - left) / 2;
        long hours = 0;
        for (int p : piles) hours += (p + mid - 1) / mid; // ceil(p/mid)
        if (hours <= h) right = mid;   // valid speed, try slower
        else left = mid + 1;           // too slow, must go faster
    }
    return left;
}

Concept questions
What two properties identify a DP problem? Optimal substructure (optimal solution built from optimal sub-solutions) and overlapping subproblems (same sub-problems computed multiple times in naive recursion).
Top-down (memoization) vs bottom-up (tabulation) — which is faster? Both are O(n) time. Bottom-up has better cache performance (sequential array access). Top-down is easier to write and only computes needed states. Either is acceptable.
How do you define dp[i] for House Robber? dp[i] = maximum money robbing from houses 0 to i. Transition: dp[i] = max(dp[i-1], dp[i-2] + nums[i]). Defines "don't rob house i" vs "rob house i plus best from two houses ago."
What is the "define dp[i] before you code" discipline? Writing dp[i] = "what this represents" before touching code forces you to think about what the index means, which makes the transition obvious. Without this, DP becomes guesswork.

// House Robber — space-optimized to O(1)
public int rob(int[] nums) {
    int prev2 = 0, prev1 = 0;
    for (int n : nums) {
        int curr = Math.max(prev1, prev2 + n);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

// Coin Change
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1);  // fill with "impossible" sentinel
    dp[0] = 0;
    for (int i = 1; i <= amount; i++)
        for (int c : coins)
            if (c <= i) dp[i] = Math.min(dp[i], dp[i-c] + 1);
    return dp[amount] > amount ? -1 : dp[amount];
}

Number of Islands · Course Schedule · Clone Graph · Pacific Atlantic
Medium ▼

Concept questions
BFS vs DFS on graphs — when to use each? BFS: shortest path (unweighted), level-by-level, minimum steps. DFS: cycle detection, topological sort, all-paths, flood fill. Both: connected components, reachability.
Why must you track visited nodes in graph traversal? Graphs can have cycles (unlike trees). Without visited tracking, traversal loops forever. Use a Set or mark grid cells in-place.
What is topological sort and when does it apply? Linear ordering of nodes where every directed edge u→v means u comes before v. Only valid for DAGs (Directed Acyclic Graphs). Use for dependency ordering (build systems, course prerequisites).
How do you detect a cycle in a directed graph? DFS with three states: WHITE (unvisited), GRAY (in current DFS stack), BLACK (done). GRAY→GRAY edge = back edge = cycle.

// Number of Islands — DFS flood-fill
public int numIslands(char[][] g) {
    int count = 0;
    for (int r = 0; r < g.length; r++)
        for (int c = 0; c < g[0].length; c++)
            if (g[r][c] == '1') { count++; dfs(g, r, c); }
    return count;
}
void dfs(char[][] g, int r, int c) {
    if (r<0||r>=g.length||c<0||c>=g[0].length||g[r][c]!='1') return;
    g[r][c] = '0';  // mark visited
    dfs(g,r+1,c); dfs(g,r-1,c); dfs(g,r,c+1); dfs(g,r,c-1);
}

8 · Heap / Priority Queue — Top-K Problems

Medium ▼

Key Code
Concept questions
Min-heap vs max-heap — which for "Kth largest"? Min-heap of size K. Poll when size > K (removes smallest). After all elements, heap contains K largest, with minimum at top = Kth largest. Counterintuitive: you use min-heap to find kth LARGEST.
What is the time complexity of heap operations? Insert (offer): O(log n). Remove min/max (poll): O(log n). Peek: O(1). Building a heap from n elements: O(n), not O(n log n) — this is the heapify property.
How do you make a max-heap in Java? PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder()); Or: new PriorityQueue<>((a, b) → b - a);
Find Median from Data Stream — why two heaps? Max-heap for lower half, min-heap for upper half. Keep sizes balanced (differ by at most 1). Median = top of max-heap (odd total) or average of both tops (even total). Each operation O(log n).

// Kth Largest — min-heap of size K
public int findKthLargest(int[] nums, int k) {
    PriorityQueue<Integer> heap = new PriorityQueue<>();
    for (int n : nums) {
        heap.offer(n);
        if (heap.size() > k) heap.poll(); // remove smallest
    }
    return heap.peek();  // min of K largest = Kth largest
}

Concept questions
What is a monotonic stack? A stack that maintains elements in either strictly increasing or decreasing order. Elements that violate the order are popped before the new element is pushed. Used for "next greater/smaller element" problems.
Monotonic decreasing vs increasing — which for Daily Temperatures? Decreasing: stack holds indices of temperatures waiting for a warmer day. When we find a warmer temp, pop all colder indices and record the distance. Push current index.
Min Stack — why do we need two stacks? Main stack: normal push/pop. Min-tracker: only push if new value ≤ current min. Pop min-tracker only when main stack pops the same value. getMin() = minStack.peek() in O(1).

10 · Backtracking — Subsets, Permutations, Combinations
Subsets · Permutations · Combination Sum · Word Search
Hard ▼
Concept Questions
Problems & Links
Key Code
Concept questions
What's the backtracking template? Choose → Explore → Unchoose. Make a choice, recurse with it, then undo the choice. This explores every branch of the decision tree while reusing the same mutable state.
Why add a COPY to results, not the list itself? The list is mutable and shared across all recursive calls. If you add the reference, by the time you read results, all entries point to the same (now-empty) list. new ArrayList<>(current) creates a snapshot.
Subsets vs Permutations — what's different in the code? Subsets: use start index (i+1) to avoid revisiting elements, collect result at every node. Permutations: no start index (can use any unused element), collect result only at leaves (when path.size() == n).
How do you prune in Combination Sum? Sort the candidates first. In the loop, if candidates[i] > remaining, break — no need to try larger values since array is sorted and remaining would only get worse.

// Subsets — the universal backtracking template
public List<List<Integer>> subsets(int[] nums) {
    List<List<Integer>> res = new ArrayList<>();
    bt(nums, 0, new ArrayList<>(), res);
    return res;
}
void bt(int[] nums, int start, List<Integer> cur, List<List<Integer>> res) {
    res.add(new ArrayList<>(cur));   // snapshot at every node
    for (int i = start; i < nums.length; i++) {
        cur.add(nums[i]);              // CHOOSE
        bt(nums, i + 1, cur, res);   // EXPLORE
        cur.remove(cur.size() - 1);   // UNCHOOSE
    }
}
